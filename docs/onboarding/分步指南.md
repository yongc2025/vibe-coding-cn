# 📘 分步指南

> 每一步告诉你：用什么、做什么、产出什么。适合第一次用 vibe-coding-cn 的人。

---

## 第 0 步：环境准备

### 确认工具

```bash
python3 --version   # 需要 3.11+
git --version
node --version      # 可选，前端项目需要
```

### 克隆仓库

```bash
git clone --recursive https://github.com/yongc2025/vibe-coding-cn.git
cd vibe-coding-cn
git submodule update --init --recursive
```

### 搭建开发环境

按顺序阅读：
1. [网络环境配置](../../assets/documents/guides/getting-started/网络环境配置.md)
2. [开发环境搭建](../../assets/documents/guides/getting-started/开发环境搭建.md)
3. [IDE 配置](../../assets/documents/guides/getting-started/IDE配置.md) 或 [OpenCode CLI 配置](../../assets/documents/guides/getting-started/OpenCode-CLI配置.md)

### 选一个 AI 工具

| 工具 | 特点 | 推荐场景 |
|------|------|---------|
| **Cursor** | IDE 集成，上手快 | 日常开发 |
| **Claude Code** | CLI 工具，上下文强 | 复杂项目 |
| **GitHub Copilot** | VS Code 集成 | 代码补全 |
| **Gemini CLI** | 免费额度 | 脚本、文档 |

**产出：** 仓库已克隆、环境就绪、AI 工具已选好。

## 第 1 步：创建项目

### 用 vibe-init.sh 初始化

```bash
./vibe-init.sh --help                    # 查看所有选项
./vibe-init.sh --type quant-crypto --name my-bot   # 创建项目
./vibe-init.sh --type quant-crypto --name my-bot --dry-run  # 先预览
```

**支持的项目类型：**

| 类型 | 说明 |
|------|------|
| `quant-crypto` | 加密货币量化（ccxt + cryptofeed + hummingbot + timescaledb） |
| `quant-astock` | A 股量化（tushare + akshare + timescaledb） |
| `quant-usstock` | 美股量化（alpaca + polygon + timescaledb） |
| `app` | APP/小程序（canvas-dev + ddd-doc-steward） |
| `enterprise` | 企业级应用（canvas-dev + ddd-doc-steward + sop-generator） |
| `saas` | 互联网 SaaS（canvas-dev + ddd-doc-steward + sop-generator） |
| `custom` | 自定义（需手动指定 --skills） |

### 填写项目定义

进入项目目录，创建 `docs/PROJECT_BRIEF.md`：

```markdown
# 项目定义

## 1. 目标：我要解决什么问题？
（一句话）

## 2. 现状：当前是什么情况？
（已有资源、数据、技术基础）

## 3. 差距：从现状到目标，缺什么？
（需要新建什么、学习什么、集成什么）

## 4. 判断标准：怎么知道做完了？
（验收条件，可测试、可度量）
```

### 用 AI 生成 README.md

把 PROJECT_BRIEF.md 喂给 AI：

```
请根据 docs/PROJECT_BRIEF.md 生成一份完整的 README.md，包含：
1. 项目名称和一句话描述
2. 技术栈选择及理由
3. 核心功能列表
4. 快速开始指南
5. 目录结构说明
```

**产出：** 项目目录已创建、Skills 已复制、PROJECT_BRIEF.md 已填写、README.md 已生成。

## 第 2 步：搭建骨架

### 选择架构模板

```bash
cat assets/documents/principles/fundamentals/通用项目架构模板.md
```

模板包含 4 种标准结构：
- **通用模板** — Flask/FastAPI 后端
- **数据科学模板** — 量化/ML/数据分析
- **前后端分离模板** — 全栈应用
- **微服务模板** — 大型项目

### 让 AI 生成目录结构

```
请参考通用项目架构模板中的 [对应模板]，
为项目 [项目名] 生成完整的目录结构。

要求：
- 包含所有 __init__.py 和 .gitkeep
- 包含 CLAUDE.md、AGENTS.md
- 包含 .gitignore、.env.example、pyproject.toml
```

### 创建配置文件

| 文件 | 用途 |
|------|------|
| `CLAUDE.md` | Claude 持久上下文（引用 Skills） |
| `AGENTS.md` | Codex 持久上下文 |
| `.gitignore` | Git 忽略规则 |
| `.env.example` | 环境变量示例 |
| `pyproject.toml` | Python 项目配置 |
| `requirements.txt` | 依赖列表 |

**产出：** 完整的目录结构、核心配置文件。

## 第 3 步：加载 Skills

### 确认 Skills 已就位

```bash
ls skills/
# 应该看到 vibe-init.sh 复制过来的 Skills
```

### 在 CLAUDE.md 中引用

```markdown
## 参考技能
@skills/ccxt/SKILL.md
@skills/postgresql/SKILL.md
@skills/canvas-dev/SKILL.md
```

### 按需添加更多 Skills

```bash
# 从母机复制额外的 Skill
cp -r ../assets/skills/telegram-dev/ skills/

# 或者用元技能生成新的
cat skills/skills-skills/SKILL.md
# 然后让 AI 根据你的领域资料生成新 Skill
```

### 查看推荐

```bash
python assets/scripts/skill-picker.py --type quant-crypto
python assets/scripts/skill-picker.py --list
python assets/scripts/skill-picker.py --search telegram
```

**产出：** Skills 已引用到 CLAUDE.md、缺少的 Skill 已补充。
