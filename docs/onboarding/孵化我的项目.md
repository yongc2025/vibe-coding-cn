# 🚀 孵化你的项目

> 你有一个想法，想用 vibe-coding-cn 把它变成代码项目？这份指南带你走完全程。

---

## 这个仓库能给你什么？

[vibe-coding-cn](https://github.com/yongc2025/vibe-coding-cn) 是一套 **Vibe Coding 工作站**。它不是一个代码库，而是一个**母机**——你从这里初始化自己的项目，把 Skills、提示词、工作流复制过去。

克隆后你拥有：

| 资源 | 路径 | 用途 |
|------|------|------|
| 🛠️ **37 个 Skills** | `assets/skills/` | 让 AI 变成领域专家 |
| 🚀 **初始化脚本** | `vibe-init.sh` | 一键创建新项目（自动复制 Skills + 配置） |
| 🔄 **全自动工作流** | `assets/workflow/auto-dev-loop/` | 需求→计划→实施→验证的 5 步闭环 |
| 📋 **提示词库** | `assets/prompts/` | 编程场景专用提示词 |
| 📂 **案例研究** | `assets/documents/case-studies/` | 真实项目的完整开发过程 |
| 📖 **方法论文档** | `assets/documents/principles/` | 核心理念、开发经验、架构原则 |

## 5 步完成你的项目

### 第 1 步：克隆仓库

```bash
git clone --recursive https://github.com/yongc2025/vibe-coding-cn.git
cd vibe-coding-cn
git submodule update --init --recursive
```

### 第 2 步：用 vibe-init.sh 创建项目

```bash
# 查看所有项目类型
./vibe-init.sh --help

# 创建项目（选一个）
./vibe-init.sh --type quant-crypto --name my-bot        # 加密货币量化
./vibe-init.sh --type quant-astock --name my-astock      # A 股量化
./vibe-init.sh --type quant-usstock --name my-usstock    # 美股量化
./vibe-init.sh --type app --name my-app                  # APP/小程序
./vibe-init.sh --type enterprise --name my-enterprise    # 企业级应用
./vibe-init.sh --type saas --name my-saas                # 互联网 SaaS
./vibe-init.sh --type custom --name my-project --skills ccxt,postgresql  # 自定义

# 带上全自动工作流
./vibe-init.sh --type quant-crypto --name my-bot --with-workflow

# 先预览不执行
./vibe-init.sh --type quant-crypto --name my-bot --dry-run
```

这个脚本会：
- 创建项目目录
- 复制对应的 Skills 到 `skills/`
- 生成 CLAUDE.md / AGENTS.md
- 生成 .codex/ 配置
- （可选）复制 auto-dev-loop 工作流

### 第 3 步：填写项目定义

进入项目目录，创建 `docs/PROJECT_BRIEF.md`：

```markdown
# 项目定义

## 1. 目标：我要解决什么问题？
（一句话说清楚）

## 2. 现状：当前是什么情况？
（已有哪些资源、数据、技术基础）

## 3. 差距：从现状到目标，缺什么？
（需要新建什么、学习什么、集成什么）

## 4. 判断标准：怎么知道做完了？
（具体的验收条件，可测试、可度量）
```

> 这是「问题求解框架」——先想清楚再动手，省掉 80% 的返工。
> 参考：[问题求解能力](../assets/documents/principles/fundamentals/问题求解能力.md)

### 第 4 步：用 AI 开发

确认 Skills 已就位：

```bash
ls skills/
# 应该看到 vibe-init.sh 复制过来的 Skills
```

在 CLAUDE.md 中引用 Skills：

```markdown
## 参考技能
@skills/ccxt/SKILL.md
@skills/postgresql/SKILL.md
```

如果缺少需要的 Skill，用元技能生成：

```bash
cat skills/skills-skills/SKILL.md
# 然后让 AI 根据你的领域资料生成新的 SKILL.md
```

**推荐开发顺序：**

```
接口定义 → 配置管理 → 核心实现 → 数据集成 → 测试验证
```

**如果用了 `--with-workflow`：**

```bash
ls workflow/auto-dev-loop/
# step1 → step2 → step3 → step4 → step5（循环控制）
```

### 第 5 步：验证与提交

```bash
# 测试
pytest tests/ -v

# 代码质量
black src/ tests/

# 提交
git add .
git commit -m "feat: 完成核心功能"
git push origin main
```

## 核心原则

1. **规划就是一切** — 先想清楚再动手
2. **上下文是第一性要素** — 垃圾进，垃圾出
3. **先结构，后代码** — 一定先规划好框架
4. **接口先行，实现后补** — 先定义输入输出
5. **一次只改一个模块** — 保持专注
6. **文档即上下文** — 不是事后补，是同步写
7. **奥卡姆剃刀** — 如无必要，勿增代码
8. **凡是 AI 能做的，就不要人工做**
